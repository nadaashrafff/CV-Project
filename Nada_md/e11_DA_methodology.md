## 1. Problem Statement

E11-original produced a two-stage detector (YOLO for localization, Hybrid SVM for classification) that achieved mAP@0.5 = 0.3840, a significant regression from E1's end-to-end YOLO at 0.4559 (Δ = -0.0719). Post-hoc analysis identified one dominant root cause: **training–inference distribution mismatch**.

### The Domain Gap

The Hybrid SVM in E11-original was trained on features extracted from **ground-truth (GT) crops** — bounding boxes drawn by a human annotator. At inference time, the same SVM was applied to features extracted from **YOLO-predicted crops** — bounding boxes produced by a detector with recall ≈ 0.51 and imperfect localization.

These two crop populations are fundamentally different:

| Property | GT Crops (training) | Predicted Crops (inference) |
|----------|--------------------|-----------------------------|
| Boundary accuracy | Perfect — drawn by annotator | Imperfect — regressed by detector |
| Content centering | Object centered in crop | Object may be off-center |
| Background contamination | Minimal (tight boxes) | Variable (loose/shifted boxes) |
| Partial objects | None | Common at image edges or occlusion |
| Multi-object crops | None (one GT box = one object) | Possible (overlapping detections) |
| Class distribution | Reflects dataset distribution | Biased by YOLO's per-class recall |

A classifier trained exclusively on GT crops learns decision boundaries calibrated to ideal, clean inputs. When it encounters noisy, imperfect predicted crops at inference, those inputs fall outside the learned distribution. The result observed in E11: **overconfident wrong predictions** — average SVM confidence on disagreements was 0.668 vs YOLO's 0.508 — and BIODEGRADABLE acting as a catch-all class for ambiguous crops.

This is a known failure mode in two-stage detection literature. It is the reason Faster R-CNN trains its classification head on Region of Interest (RoI) proposals generated by the Region Proposal Network, **not** on GT boxes. Training on proposals exposes the classifier to the same imperfect crop distribution it will face at inference.

---

## 2. Research Question

> Does retraining the Hybrid SVM on features extracted from YOLO-predicted crops (rather than GT crops) close the domain gap and recover detection mAP?

Two sub-questions:

1. **Improvement magnitude**: By how much does mAP improve, and is that improvement enough to exceed E1 (0.4559)?
2. **Agreement structure**: Does the domain-adapted SVM agree more with YOLO's class predictions, and does higher agreement correlate with higher mAP?

## 3. Method: Domain-Adapted Training

### 3.1 High-Level Pipeline

```
TRAINING PHASE (one-time, on 70% train split)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  7,324 training images
       │
       ▼
  YOLOv8n predict()                      [conf=0.25, iou=0.45 — same as E1]
       │
       ├──────────────────────────────────────────────────────┐
       ▼                                                      ▼
  Predicted boxes                                   GT boxes YOLO MISSED
  IoU ≥ 0.50 with any GT?                          (~50% of all GT objects)
    YES → matched_sample (source="predicted")                 │
    NO  → discard (false positive)                            ▼
       │                                            Jitter Fill (Cell 7b)
       │                                            translate ±20% box size
       │                                            scale × U[0.80, 1.25]
       │                                            aspect × U[0.90, 1.10]
       │                                            keep if IoU ∈ [0.50, 0.80]
       │                                                      │
       │                                            jitter_sample (source="jitter")
       │                                                      │
       └────────────────────┬─────────────────────────────────┘
                            ▼
               merged_samples (~45K, all noisy)
                            │
  Each sample (predicted OR jitter — identical code path):
    ├── Crop bbox from full image
    ├── Resize 224×224 (BILINEAR)
    ├── Extract SPPF 256-dim
    └── Extract Classical 252-dim
                            │
                            ▼
              N × 508 feature matrix + GT labels
                            │
  Per-modality StandardScaler → SVM(RBF, C=10) → Domain-Adapted SVM  ←── save


INFERENCE PHASE (test set, unchanged from E11-original)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Test image
    │
    ▼ YOLO → predicted boxes + YOLO confidence
    │
  For each box:
    ├── Crop + resize 224×224
    ├── Extract SPPF (256) + Classical (252)
    ├── Scale with DA scalers
    └── DA-SVM classifies → class label
    │
    ▼  Store: class=DA-SVM, confidence=YOLO_detection_conf
    │
  Compute mAP@0.5 (PASCAL VOC)
```

### 3.2 Why IoU ≥ 0.50 as the Match Threshold

The PASCAL VOC evaluation protocol defines a detection as "correct" only if its IoU with a GT box exceeds 0.50. Using the same threshold for label assignment ensures internal consistency: we only train on predicted boxes that, by the evaluation protocol's own standard, are valid detections of real objects. A predicted box with IoU 0.40 could be assigning a noisy partial crop to a GT label, poisoning the classifier with highly ambiguous training examples.

### 3.3 Crop Preprocessing — Why It Must Match Inference Exactly

The domain adaptation only works if the feature extraction pipeline used during training is **byte-for-byte identical** to the pipeline used during inference. Both use:

1. OpenCV `cvtColor(BGR → RGB)` on the full image
2. Pixel clamp to image bounds: `x1 = max(0, int(x1))`
3. Resize with `cv2.INTER_LINEAR` to 224×224
4. SPPF: `T.ToTensor()` (÷255 only, no ImageNet normalization) → `nn_backbone` forward → GAP on layer-9 output
5. Classical: identical to `src/features/classical_features.py`

Any deviation — a different resize method, different normalization, different crop order — reintroduces distribution shift even while "domain-adapting."

### 3.4 Why the Confidence Score Remains YOLO's

mAP ranking requires a confidence score per detection to sweep precision–recall thresholds. Using YOLO detection confidence (not SVM class probability) is deliberate:

- YOLO confidence reflects **localization quality**: how certain YOLO is that a real object is present in this region.
- SVM probability reflects **classification confidence**: how certain the SVM is about which class the crop belongs to.
- For mAP, ranking by localization quality is the correct signal — a high-confidence detection that is misclassified contributes a false positive regardless of SVM confidence.
- Using YOLO confidence keeps E1, E11-original, and E11-DA on an equal footing for ranking. The only variable that changes across these experiments is the source of the class label.

## 5. Expected Training Set Statistics

YOLO's recall on this dataset is approximately 0.50 (E1 result: 0.5063). With IoU ≥ 0.50 matching:

- Total GT annotations in train split: ~45,177
- Expected YOLO predictions on training images: ~50,000–60,000 (some images have multiple predicted boxes per object due to NMS proximity)
- Expected matched predictions (IoU ≥ 0.50): ~22,000–28,000

The matched set will be **smaller** than the GT crop set and **class-imbalanced** differently from the original dataset — YOLO's per-class recall varies (METAL: 0.5792, BIODEGRADABLE: 0.5152, GLASS: 0.4808 in E1), so some classes will be underrepresented in the domain-adapted training set. The `class_weight='balanced'` SVM parameter compensates for this.

## 6. Jitter Fill — Closing the Coverage Gap

### 6.1 The Problem Jitter Solves

E11-DA Stage 1 only recovers predicted boxes that overlap a GT box at IoU ≥ 0.50. Because YOLO's recall on this dataset is ≈ 0.50, approximately **half of all GT objects are never detected** during training-set inference. These missed GT boxes leave systematic holes in the domain-adapted training distribution: the SVM never sees any training examples for those specific objects, even in their noisy predicted-crop form.

The practical consequence is that the DA training set shrinks from ~45K (original GT crops) to ~22–28K (matched predicted crops), and the class composition shifts according to YOLO's per-class recall rates — classes that YOLO misses more often are underrepresented, partially defeating the purpose of adaptation.

### 6.2 Bounding Box Jitter

For each GT box that YOLO failed to detect, we synthetically perturb it to simulate what a real (imperfect) detector might have predicted. This is called **bounding box jitter**, and is the same principle used in:
- **Faster R-CNN** training, where the classification head is trained on RoI proposals rather than GT boxes
- **Hard negative mining** in DPM and R-CNN, where the trained detector's own outputs are recycled as training examples
- Data augmentation pipelines in modern detection frameworks (e.g. Mosaic, Copy-Paste)

### 6.3 Jitter Parameters and Calibration

The perturbation must be calibrated to produce crops whose content and spatial noise resemble real YOLO-predicted crops. We target IoU 0.50–0.80 with the original GT box — matching the typical IoU range of YOLO's true positive detections.

Three independent perturbations are applied:

| Perturbation | Range | Simulates |
|-------------|-------|-----------|
| Center translation | ±20% of box width/height | YOLO localization error — center drift |
| Scale factor | Uniform[0.80, 1.25] | YOLO size estimation error — over/under-detection |
| Aspect ratio | Uniform[0.90, 1.10] | YOLO shape fitting error — mild width/height imbalance |

Each jitter attempt is accepted only if the resulting box satisfies IoU ∈ [0.50, 0.80] with the original GT box. If no valid jitter is found within 40 attempts (e.g. for very small GT boxes near image edges), the sample is discarded rather than forcing a bad crop.

### 6.4 Why 1 Jitter Per Missed GT Box

Generating exactly one jitter per missed GT box keeps the predicted and jittered contributions balanced (approximately 50/50). Generating more jitters per box would bias the training set toward jittered crops, which carry synthetic rather than real noise — the opposite of what we want. One-to-one keeps the real predicted crops as the dominant signal while filling coverage gaps.

### 6.5 Checkpoint Position

The jitter step runs **after** Stage 1 and **before** feature extraction. This means:
- If Stage 1 checkpoint (`da_matched_train.json` + `da_unmatched_gt.json`) exists, jitter runs in seconds (pure numpy, no I/O)
- If jitter checkpoint (`da_matched_merged.json`) exists, feature extraction (Cell 8) uses the merged set directly
- Each stage is independently resumable

### 6.6 Impact on Training Set

| Source | Approximate Count | Noise Type |
|--------|-------------------|------------|
| YOLO-predicted matched | ~22,000–28,000 | Real detector localization error |
| Jitter fills (missed GT) | ~17,000–23,000 | Synthetic perturbation, calibrated to IoU 0.50–0.80 |
| **Total merged** | **~45,000** | Mixed |

The merged set approximates the original GT crop count while ensuring the SVM sees noisy, boundary-imperfect crops for all GT objects — not just the half that YOLO happened to detect.


## 7. Checkpointing Strategy

Feature extraction over ~45,000 merged crops takes approximately 20–35 minutes (classical Gabor filters dominate). The code implements three-level checkpointing:

**Checkpoint 1 — `da_matched_train.json` + `da_unmatched_gt.json`**  
Saved after YOLO inference on all 7,324 training images. Both files must exist to skip Stage 1. Contains predicted matched crops and a list of YOLO-missed GT boxes waiting for jitter.

**Checkpoint 2 — `da_matched_merged.json`**  
Saved after jitter fill. Contains the merged set: predicted crops + jitter fills. If this file exists, Cell 7b is skipped and Cell 8 goes directly to feature extraction on the merged set.

**Checkpoint 3 — `da_yolo_train_X.npy` / `da_classical_train_X.npy` / `da_train_y.npy`**  
Saved after feature extraction. If these three files exist, feature extraction is skipped and SVM training proceeds immediately.

The chain: Cell 7 → Cell 7b → Cell 8 → Cell 9. Any cell can be the entry point on restart — zero repeated work upstream.

## 8. Possible Outcomes and Interpretations

### Outcome A: E11-DA > E1 (mAP > 0.4559)
Domain adaptation fully closes the gap. The hybrid classifier, when trained on the same type of noisy crops it sees at inference, can outperform end-to-end YOLO. This validates the two-stage architecture as a viable alternative when the classification component has domain-matched training data.

**Report framing**: "Domain-adapted hybrid classification surpasses end-to-end YOLO, demonstrating that training–inference distribution alignment is the critical design factor in two-stage detection."

### ✅ Outcome B (OCCURRED): E11-orig < E11-DA ≤ E1 (0.3840 < 0.3897 ≤ 0.4559)
Domain adaptation partially closes the gap. The DA-SVM recovers +0.0057 mAP over E11-original, but end-to-end YOLO's joint optimization of localization and classification holds a −0.0662 advantage that domain alignment alone cannot close.

**Report framing**: "Domain adaptation recovers +0.0057 mAP points from the domain-naive baseline, demonstrating that GT–predicted crop distribution shift is a significant but not exhaustive source of performance degradation in decoupled two-stage systems. The residual gap of −0.0662 vs E1 reflects a structural bottleneck inherent to architectural decoupling of localization and classification."

### Outcome C: E11-DA ≈ E11-orig (no significant improvement)
Domain adaptation does not help. This would indicate that:
1. The domain gap is not the primary failure mode
2. The SVM architecture itself (linear kernel in high-dim space) is the bottleneck
3. YOLO's features from predicted crops carry insufficient discriminative information even when the classifier is trained on the same distribution

**Report framing**: "Retraining on domain-matched predicted crops does not close the performance gap, suggesting that the bottleneck lies in the feature discriminability of predicted crops rather than in training distribution mismatch."

## 9. Connection to Related Work

The domain shift problem in two-stage detectors is not new:

- **Faster R-CNN (Ren et al., 2015)** explicitly trains the RoI classification head on RoI proposals rather than GT boxes, using a 1:3 positive-to-negative ratio.
- **Hard Negative Mining** in earlier frameworks (DPM, R-CNN) runs the trained classifier on the training set itself to find false positives and re-train with them — an analogous approach.
- **Domain Adaptive Faster R-CNN (Chen et al., 2018)** addresses a different type of domain shift (source/target domains) but the principle of aligning training distribution to inference distribution is the same.

E11-DA applies the same principle in a lightweight way: instead of modifying the detector architecture, we simply regenerate the training data for the classification head using the detector's own predictions.

|--------|--------|---------|-----|------------------------|------------------|

| Total test detections | 8,341 | 8,341 | — | — | — |
| YOLO/SVM agreement rate | 85.3% | 85.2% | — | +0.1 pp | — |
| Val macro-F1 (GT crops) | 0.7215 | 0.7345 | — | −0.0130 | — |

The agreement rate is essentially unchanged (+0.1 percentage points), confirming that the DA-SVM has not simply learned to parrot YOLO's class assignments — it has independently reorganized its decision boundaries to match the noisy-crop distribution, yet the two models still largely agree on class label, with the small agreement rate gain driven by better METAL and PAPER classification.

Val macro-F1 drops slightly (0.7215 vs 0.7345) because the DA-SVM is now evaluated on GT crops at val time — a distribution it was NOT trained on. This is the expected cross-distribution penalty and does not indicate overfitting.

### 10.2 Per-Class AP

| Class | E11-DA | E11-Orig | E1 | Δ (E11-DA vs E11-orig) | Δ (E11-DA vs E1) |
|-------|--------|---------|-----|------------------------|------------------|
| BIODEGRADABLE | 0.4445 | 0.4441 | 0.4663 | +0.0004 | −0.0218 |
| CARDBOARD | 0.3920 | 0.3915 | 0.4445 | +0.0005 | −0.0525 |
| GLASS | 0.3898 | 0.3826 | 0.4575 | **+0.0072** | −0.0677 |
| METAL | 0.4208 | 0.4065 | 0.5161 | **+0.0143** | −0.0953 |
| PAPER | 0.3479 | 0.3256 | 0.3939 | **+0.0223** | −0.0460 |
| PLASTIC | **0.3435** | **0.3537** | 0.4572 | **−0.0102** ⚠ | −0.1137 |

**PAPER gains the most** (+0.0223): YOLO's localization errors for flat, thin objects (paper sheets) produce crops that are particularly different from GT crops — DA training gives the SVM the right prior for these noisy boundaries.

**METAL gains significantly** (+0.0143): Similar reasoning — metallic surfaces cause YOLO to drift slightly off-center; GT training made the SVM over-confident on perfectly framed crops.

**PLASTIC degrades** (−0.0102): This is the most important anomaly. PLASTIC is visually the most heterogeneous class — bottles, bags, containers, films. The DA training set for PLASTIC carries real YOLO-prediction noise but the class was underrepresented (8.8% of the DA set, even with `class_weight='balanced'`). Additionally, BIODEGRADABLE's severe over-representation (61.3%) in the matched predictions biases the decision boundary — PLASTIC vs BIODEGRADABLE confusions increase. End-to-end YOLO (E1) handles PLASTIC best (AP=0.4572); the two-stage architecture consistently struggles here regardless of domain alignment.

**METAL vs E1 gap is the largest** (−0.0953): METAL requires precise localization to distinguish cans/foil from cardboard containers. YOLO's E1 joint training optimizes for this; the two-stage SVM cannot recover from imperfect box regression.

BIODEGRADABLE is severely over-represented in YOLO's predictions (61.3% vs ~49% in ground truth). This occurs because BIODEGRADABLE objects dominate the dataset and YOLO has highest recall for this class — it over-fires on BIODEGRADABLE. The `class_weight='balanced'` parameter partially compensates, but cannot fully correct for a 10:1 class ratio within the DA training set itself.

### 10.4 Key Finding

> E11-DA tests whether retraining the Hybrid SVM on features extracted from YOLO-predicted crops (domain adaptation) can close the performance gap between the two-stage detector (E11-orig: mAP@0.5 = 0.3840) and end-to-end YOLO (E1: mAP@0.5 = 0.4559). With jitter fills for undetected GT boxes, the DA-SVM achieves **mAP@0.5 = 0.3897**, recovering **+0.0057 out of the −0.0719 gap** — an 8% gap closure. The bottleneck is architectural, not distributional. PLASTIC AP actually degrades (−0.0102) due to BIODEGRADABLE over-representation in YOLO's detections biasing the DA training set. The agreement rate between YOLO and DA-SVM (85.3%) is essentially unchanged from E11-original (85.2%), confirming the failure mode is structural: decoupling localization and classification removes the gradient signal that allows E1's joint optimization to allocate feature capacity toward the hardest class boundaries. Domain adaptation is a necessary but insufficient fix for the two-stage architecture.

### 10.5 What E11-DA Proves

| Hypothesis | Verdict | Evidence |
|-----------|---------|---------|
| Domain mismatch is the primary failure mode of E11-orig | ❌ Refuted | +0.0057 closes only 8% of the −0.0719 gap |
| DA training improves mAP vs GT-SVM training | ✅ Confirmed (weakly) | E11-DA (0.3897) > E11-orig (0.3840) |
| Jitter fills help (vs pure predicted-only DA) | ✅ Plausible | Training set grew 32K → 52K; PAPER and METAL gains suggest better coverage |
| End-to-end YOLO is architecturally superior | ✅ Confirmed | E1 leads by −0.0662 after full DA treatment |
| PLASTIC is the structural weak point | ✅ Confirmed | Worst DA degradation (−0.0102), worst E1 gap (−0.1137) |

| Rank | Experiment | Architecture | mAP@0.5 |
|------|-----------|-------------|---------|
| 🥇 1 | E1 YOLO Baseline | End-to-end YOLOv8n | **0.4559** |
| 🥈 2 | E11-DA | YOLO + DA-SVM (predicted crops) | **0.3897** |
| 🥉 3 | E11-orig | YOLO + GT-SVM (clean crops) | **0.3840** |

E11-DA holds second place in the detection track, validating that domain adaptation is a meaningful improvement over the naive two-stage baseline — but the architectural ceiling of decoupled detection remains well below end-to-end joint optimization.
