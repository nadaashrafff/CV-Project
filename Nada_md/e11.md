## 1. Research Question

> Can combining YOLO's detection-optimized features with classical hand-crafted features produce a classifier that is more accurate than either modality alone, and can this hybrid approach improve YOLO's detection confidence when plugged back into the full-image pipeline?

---

## 2. Motivation

E11 is the only experiment in the project that bridges both evaluation tracks:

| Track | Experiments | Purpose |
|-------|------------|---------|
| Detection Track | E1, **E11** | Full image detection (mAP@0.5) |
| Classification Track | E2–E10, **E11** | Crop-level feature representation analysis (Macro-F1) |

**Why this matters:**
- **YOLO (E1)** is great at *finding* objects (bounding boxes) but sometimes misclassifies them. Its SPPF features capture spatial context and scene-level semantics but lack explicit material descriptors.
- **Classical features (E2)** are excellent at describing *what something looks like* (color, texture, edges) but have no understanding of where the object is in the image or its spatial context.
- **E11 combines both** to create a "two-stage" system: YOLO finds the objects, then the hybrid classifier re-evaluates what they actually are.

## 3. Feature Pipeline

### 3.1 Input Features

| Feature Source | Dimensions | Extraction Method | What It Captures |
|---------------|-----------|-------------------|------------------|
| YOLO SPPF | 256 | Forward hook on Layer 9 of YOLOv8n, Global Average Pooling over spatial dims `[B, 256, H, W] → [B, 256]` | Scene context, object shape, spatial relationships |
| Classical | 252 | Hand-crafted pipeline (Color histograms 96 + LBP 26 + Gabor 48 + HOG 36 + Shape 10 + GLCM 36) | Explicit texture, color distribution, edge patterns |
| **Fused** | **508** | `np.hstack([yolo, classical])` after StandardScaler | Best of both worlds |

### 3.2 Fusion Strategy: Early Fusion (Concatenation)

```
YOLO features (256-dim) ──→ StandardScaler ──┐
                                              ├── np.hstack ──→ 508-dim vector ──→ Classifier
Classical features (252-dim) ──→ StandardScaler ──┘
```

**Why StandardScaler before fusion:**
- YOLO features and classical features live on completely different numerical scales.
- Without normalization, one branch would dominate the other based purely on magnitude, not information content.
- StandardScaler ensures both branches have mean=0, std=1, giving the classifier equal access to both.

## 4. Classification Models Tested

### 4A. Multi-Layer Perceptron (MLP) — PyTorch

| Parameter | Value |
|-----------|-------|
| Architecture | Linear(508→256) → BN → ReLU → Dropout(0.3) → Linear(256→128) → BN → ReLU → Dropout(0.2) → Linear(128→6) |
| Optimizer | Adam (lr=1e-3, weight_decay=1e-4) |
| Loss | CrossEntropyLoss with balanced class weights |
| Scheduler | ReduceLROnPlateau (factor=0.5, patience=5) |
| Early Stopping | Patience=10, monitoring val Macro-F1 |
| Batch Size | 256 |
| Total Parameters | ~166,790 |

**MLP Results:**
- Training time: **17 seconds** on CPU
- Best val Macro-F1: **0.7124**
- Early stopped at epoch ~28

### 4B. Support Vector Machine (SVM) — scikit-learn

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| Kernel | RBF | Same as E3 and E5 (proven most effective) |
| C | 10.0 | Regularization — same as E3/E5 for fair comparison |
| gamma | "scale" | Default, adapts to feature variance |
| class_weight | "balanced" | Handles the 10.3:1 class imbalance (BIODEG vs CARDBOARD) |
| probability | True | Enables `predict_proba()` for AUC-ROC and downstream fusion |
| random_state | 42 | Locked project seed |

**Why SVM over MLP:**
- Neural networks can overfit on flat, tabular feature vectors (as seen in E10 Attention Fusion, which underperformed E5 PCA+SVM).
- The RBF SVM consistently outperformed other classifiers in this project (E3 and E5 are both SVM-based and are the two best baselines).
- Training is slower (~10 min on CPU) but produces a more stable decision boundary.

### 5.4 Key Findings (Classification Track)

**Finding 1: YOLO SPPF features are weaker than EfficientNet features for pure classification.**
- E3 (Deep SVM, 1280-dim EfficientNet) = 0.7848
- E11 (Hybrid SVM, 256-dim YOLO + 252-dim Classical) = 0.7345
- The 1280-dim EfficientNet features capture richer visual representations than the 256-dim YOLO SPPF features. This is expected because EfficientNetB0 was specifically designed for image classification (ImageNet), while YOLO's SPPF layer is optimized for object detection (localization + classification jointly).

**Finding 2: Hybrid fusion still adds significant value over classical features alone.**
- E2 (Classical only) = 0.6476
- E11 (YOLO + Classical) = 0.7345
- The +0.087 improvement proves that YOLO SPPF features carry complementary information that classical features alone cannot capture.

**Finding 3: AUC-ROC is strong (0.9452) despite lower Macro-F1.**
- This means the SVM produces well-calibrated probability estimates.
- The model "knows what it doesn't know" — it assigns low confidence to hard classes (Glass, Plastic).
- This is excellent for the Detection Track, where confidence thresholding determines which boxes survive NMS.

**Finding 4: The bottleneck is YOLO feature dimensionality, not the classifier.**
- E5 used PCA to compress 1532-dim to 939-dim and got 0.8071.
- E11 only has 508-dim input (256 YOLO + 252 Classical).
- The YOLO SPPF layer outputs only 256 dimensions — half the information budget of the classical features alone.
- If YOLO features were 512+ dimensions (e.g., from a larger YOLOv8-small), E11 would likely close the gap with E5.

## 6. Track 2: Detection Results (PENDING ⏳)

### 6.1 Pipeline Design

```
Full test image
    ↓
YOLOv8n predicts bounding boxes (same as E1)
    ↓
For each predicted box:
    ├── Extract 256-dim YOLO SPPF features (forward hook)
    ├── Crop the image region
    ├── Extract 252-dim classical features (HOG, LBP, etc.)
    ├── StandardScaler normalize both
    ├── Concatenate → 508-dim
    └── E11 SVM predicts class + confidence
    ↓
Override YOLO's class label with SVM prediction
    ↓
Compute mAP@0.5 (compare vs E1 baseline: 0.4559)
```
